EMPATHIZE

THE COMPLEX WORLD OF QUILTS

The crafting of quilts is not popular everywhere, so it is worth explaining what it is before diving deeper.

Quilting is the craft of stitching (by hand or machine) differents pieces of fabric together, with a layer of batting and a backing fabric to make a padded textile. They can have a practical use, like bedcovers, or they can be purely ornamental.

Quilting is extremelly popular in North America because of different factors:
  1. Tradition and heritage: has been part of American life since colonial times. Families passed down quilts through generations as treasured heirlooms.
  2. Community: historically, women gathered to sew together, socialize, and support each other. Modern communities continue this social aspect.
  3. Resourcefulness: a quilt often tells family stories or a major life events, and its recognized as an art form.
  4. Personal expression: originally quilts were made from leftover fabric scraps. The sense of making something beautiful out of simple materials still appeals today.

INTENDED USE

The Quilting app, unlike the crafting or embroidery one, is not directly linked to a physical company product. The app can be used as an assistant to create the quilt in a traditional hand-sewn fashion. Of corse the sewing machines and crafting machine sold by SVP can be used in the process, but the app can also be handy without owning any machine.

The Quilt app must have two sides:
  1. Plan and Create: It will allow the user on MVP to load preset quilt layouts and insert inside each block of the quilt any block design from the preset library. The user can specify the dimensions of the Quilt as a whole or the blocks, add borders and sashing, modify the coloring of the design. All this must be done with easy and in a way that results ammusing.
  2. Making it real: once the user is happy with the final design, he can pass to the "Make the Quilt" process. He will be able to check the build instrructions for each individual block design used. He can also check a map of the quilt project where the parts are displayed in different groups (pieces, blocks, wors and quilt). He will also have access to the estimated yardage estimation for each fabric used. The goal of all this info is to reduce to a minimum the chances of error and frustration.

OTHER APPS

As part of my preliminary research I wanted to have a look at what Quilt design products were already available in the market, such as Electric Quilt or PreQuilt, and their reputation amongst users. I did not really want to get deeply into them, as did not want to influence my design.

Each of the app and tools I checked had a good user base, but none was intuitive or easy enough to be the best of all. Each tool had their flaws, and by analizing them I could figure out the needs of the user:

  • Avoid Complexity: creation and customization of quilt projects must be friendly, easy and funny.
  • Too much information to manage: historically, women gathered to sew together, socialize, and support each other. Modern communities continue this social aspect.
  • Resourcefulness: getting the design right and in the right scale can be tricky. I would fix this.
  • Personal expression: the app must help the user to reduce waste of fabric.
Problem Statement:
How can we build a satisfactory Quilting experience where

We find that our main goals for the quilting app must be:
Be able to build a quilt from the start in a hassle-free funny way.

IDEATION

MODULAR BUILD

For this project I was handed a full set of technical requirements from the Product Owner at the start. On this documentation the overall method of creating a project was mostly figured out.

Essetially, a basic quilt project could be broken appart like this:

1. Quilt
Its the project as a whole. It is essentially a grid of blocks..

2. Quilt Block
Each cell within the Quilt grid can be empty or contain a design.

3. Block Design
A pattern made of sewn pieces.

4. Pieces
Each of the atoms that make a block design.

5. Border and Sashing
Optional parts that might be used.

Now we know the parts of the quilt, we can understand the modular building method on the app:

  • The user chooses a layout for the quilt grid, the number of divisons.
  • The user selects each block and assigns designs to it.
  • The color for pieces can to customized as the user desires.
  • Extra elements like border or sashing can be used.

Instructions

Each type block design has a way to be built . All these process must be included as additional documentation that the user can inspect at any point.

In order to get this part of the app right, the collaboration of the quilt expert was fundamental: he provided the list of the best quiltblock designs to include for the first release, and provided all the info on how they are called and assembled.

Yardage

The yardage page was probably the most difficult to implement and desging. Using mathematical routines, all the pieces with the same color are arranged together in a way that they are saving fabric material. To do so, triangles will rotate and shapes will stick to each other, making strips of fabric that later will be much easier to cut (manually or with the machine).Aditionally, the total required size will be visible, so the user knows the ammount of fabric beforehand.

TESTING

Once I had ready the first functional prototype, I got ready for some usability testing.

I gathered a small focus group of new and experience quilters. The tests were done remotely, and the tasks requested to perform were:
  1. Open a 3x3 quilt template from the home page.
  2. Select and assign a design to all the the blocks in thew quilt.
  3. Select and assign a new design to the central block design.
  4. Change the color fabrics in the project as desired.
  5. When happy, inspect the info on how to build the Quilt.

KEY OBSERVATIONS

Drag and drop

Several user assumed they could drag and dropquilt blocks to swap positions. This is something that will be researched and implemented in the future.

Fabric selection

Two users instinctively double-tapped fabric swatches expecting them to apply immediately. The extra “Apply” step felt unnecessary.

Export flow

The “Make the Quilt” button caused hesitation. Beginners thought it meant printing, or that it would trigger some sort of inminent process, whil experienced quilters expected the step-by-step set of information.

Onboarding

Users who skipped onboarding later said they couldn’t remember what some icons meant. The toolbar for Quilting, not having the buttons nested in dropdowns, are lacking text labels.

LEARNINGS

While working on the Quilting app, none of the previous app projects (Embroidery and Crafting) were closed, as we kept on adding post MVP features in a regular basis. I discovered how tricky it was going to be managing the UX for the three projects at the same time. The most difficult part was to sync changes. For example, we decided to change the naming for a feature that was present in all three apps;hat meant writing Jira tickets for all apps. I always had to deal with a particular app keeping the other 2 in mind.

One of my main goals with sharing the basic structure and design of the initial app on the other rest, was to save development time. I was hoping that a set of components could be created and reused for the different projects. But I did find that this was only going to be possible if the development teams were really in line, and it did not always happen. I did find that, for example the home page, was created three different times, when the initial Embroidery one could have been adapted and re-used.

Looking back I realise how important it is to be sourrounded by the right people. The assistance of a quilt specialist on the project really made things smooth, and saved me figuring out many things. He was a great source of first hand information that several times provided the right answer and did allow the project to move fordward.
